# Contributing to PearlPool

Thanks for your interest in contributing! Here's how to get started.

## Development Setup

```bash
git clone https://github.com/EasyPoolPearl/pearlpool.git
cd pearlpool
node test.js  # Run smoke tests
```

## Code Style

- Plain Node.js (no TypeScript, no npm dependencies)
- Use `require()` for imports
- JSDoc comments on all public functions
- 2-space indentation

## Pull Requests

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/my-thing`)
3. Make your changes
4. Run `node test.js` to verify tests pass
5. Commit with a clear message
6. Open a PR against `master`

## Reporting Issues

Use GitHub Issues for bug reports and feature requests. Include:
- Node.js version (`node --version`)
- OS and architecture
- Steps to reproduce
- Expected vs actual behavior

## Discussions

Use GitHub Discussions for questions, ideas, and showing off your pool setup.
